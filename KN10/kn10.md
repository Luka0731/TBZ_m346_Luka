## A) Kostenrechnung IAAS - Rehosting
### AWS
[Image](./images/a-aws.jpg)
[Click here to get to the .pdf file with more information about the estimation.](./resources/estimate-aws.pdf)
Für den Web Server wurde die t3.small Instance gewählt, da sie mit 2 GB RAM die Anforderung exakt erfüllt. Dies stellt jedoch kein Problem dar, da die zusätzliche vCPU nur minimale Mehrkosten verursacht und etwas Reserve für Lastspitzen bietet.
Der DB Server wurde mit einer t3.medium Instance konfiguriert, welche mit 2 vCPUs und 4 GB RAM den Anforderungen entspricht. Für den Speicher wurden 100 GB EBS Storage hinzugefügt.
Für die Backups wurde Amazon S3 gewählt. Die Backup-Strategie erfordert 14 gleichzeitige Kopien (7 tägliche, 4 wöchentliche, 3 monatliche), was bei 100 GB Datenbankgrösse etwa 1.4 TB Speicher ergibt.
Als Region wurde Europe (Zurich) gewählt um niedrige Latenzzeiten für Schweizer Benutzer zu gewährleisten. Das On-Demand Pricing-Modell wurde verwendet, da es sich um eine Evaluation handelt.
### Azure
[Image](./images/a-azure.jpg)
[Click here to get to the exel file with more information about the estimation.](./resources/estimate-azure.xlsx)
Für den Web Server wurde eine B2ms VM gewählt, welche mit 1 vCPUs und 2 GB RAM die Anforderungen erfüllt. Der Storage wurde auf 20 GB konfiguriert.
Der DB Server wurde mit einer B2s VM konfiguriert, welche mit 2 vCPUs und 4 GB RAM den Anforderungen entspricht. Für den Speicher wurden 100 GB hinzugefügt.
Für die Backups wurde Azure Blob Storage mit dem Cool-Tier gewählt. Cool Storage eignet sich gut für Backups, da diese nicht ständig abgerufen werden, aber bei Bedarf trotzdem schnell verfügbar sind. Die benötigten 1400 GB ergeben sich aus der Backup-Strategie (7 tägliche, 4 wöchentliche, 3 monatliche Backups × 100 GB Datenbankgrösse).
Als Region wurde Switzerland North gewählt um niedrige Latenzzeiten und Datenschutzkonformität zu gewährleisten.

## B) Kostenrechnung PAAS - Replattforming 
[Image](./images/dyno-1.jpg)
[Image](./images/dyno-2.jpg)
Für die Web-Applikation wurde ein Standard-1X Dyno ($25/mo) gewählt. Für die Datenbank wurde Heroku Postgres Standard-0 (4 GB RAM, 64 GB Storage, $50/mo) gewählt, was der RAM-Anforderung von 4 GB entspricht. Der Storage weicht von den benötigten 100 GB ab, da der nächstgrössere Plan (Standard-2 mit 256 GB für $200/mo) überdimensioniert wäre. Backups sind bei Heroku Postgres inklusive, ein separater Backup-Speicher ist daher nicht nötig.
